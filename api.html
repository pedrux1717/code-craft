System Metrics /api/system

Endpoint per la ricezione e consultazione delle metriche di sistema inviate dagli agent.

POST /api/system Invia metriche base

Riceve le metriche base da un agent: CPU, RAM, hostname. Usato per il monitoraggio essenziale.

Body (JSON)

json
{ "cpuPercent": 45.5, "ramPercent": 60.2, "hostname": "prod-01" }
POST /api/system/advanced Invia metriche avanzate

Riceve il set completo di metriche: CPU, RAM, disco, rete, load average, processi. Utilizzato dall'Agent v2.

Usato da Agent v2
GET /api/system/latest Ultimi dati di sistema

Restituisce le metriche più recenti da tutti gli host. Usato dalla Dashboard per aggiornamento real-time ogni 10 secondi.

Usato da DashboardUsato da AI Engine
GET /api/system/host/{hostname} Metriche per host specifico

Restituisce lo storico delle metriche per un singolo host. Usato dall'AI Engine per analisi su 24 ore.

GET /api/system/stats Statistiche generali

Statistiche aggregate su tutti gli host: numero metriche, host attivi, valori medi CPU/RAM.

Agents /api/agent

Gestione degli agent connessi. Un agent diventa Offline se non invia heartbeat per più di 5 minuti.

POST /api/agent/heartbeat Heartbeat agent

Segnala che l'agent è ancora attivo. Se l'agent non esiste viene creato automaticamente. Aggiorna LastHeartbeat e Status = "Online".

json
{ "hostname": "prod-01", "agentVersion": "2.0" }
GET /api/agent Lista tutti gli agent

Lista tutti gli agent con stato, ultimo heartbeat e metriche associate. Aggiorna automaticamente lo status a Offline se il heartbeat manca da più di 5 minuti.

GET /api/agent/{hostname} Dettagli agent specifico

Restituisce stato e dettagli di un agent specifico. 404 se l'hostname non esiste.

DEL /api/agent/{hostname} Elimina agent

Rimuove un agent dal database. Operazione permanente.

Alerts /api/alert

Sistema di alerting con tre livelli di severità: CRITICAL, WARNING, INFO. Gli alert vengono generati dall'AI Engine o manualmente.

POST /api/alert Crea nuovo alert

Crea un alert. Generato automaticamente dall'AI Engine quando rileva anomalie o trend critici.

json
{
  "hostname": "prod-01",
  "alertType": "HIGH_CPU",
  "message": "CPU al 92% da 5 minuti",
  "severity": "CRITICAL"
}
GET /api/alert Lista alert (con filtri)

Lista alert con supporto filtri. Query param: ?resolved=false per solo alert attivi.

GET /api/alert/host/{hostname} Alert per host

Tutti gli alert (attivi e risolti) per un host specifico.

PUT /api/alert/{id}/resolve Risolvi alert

Marca un alert come risolto. IsResolved = true, ResolvedAt = DateTime.UtcNow.

Automation /api/automation

Gestione delle azioni di self-healing. Le azioni vengono create dall'Automation Engine e tracciate nel loro ciclo di vita: PENDING → EXECUTING → DONE/FAILED.

POST /api/automation/action Crea azione di automazione

Registra una nuova azione. Tipi: KILL_PROCESS, RESTART_SERVICE, SCALE_UP, SCALE_DOWN, CLEAR_CACHE, ESCALATE.

GET /api/automation/actions Storico azioni

Lista completa di tutte le azioni eseguite, con status e timestamp. Usato dalla Dashboard per mostrare la history.

GET /api/automation/pending Azioni in attesa

Restituisce tutte le azioni con status PENDING. Usato dall'Automation Engine per sapere cosa deve ancora eseguire.

PUT /api/automation/action/{id}/status Aggiorna status azione

Aggiorna lo status di un'azione (PENDING → EXECUTING → DONE/FAILED). Permette il tracking del ciclo di vita completo.

Quick Start

Test metriche

Invia una metrica di test al backend con curl.

bash
curl -X POST \
  http://localhost:5000/api/system \
  -H "Content-Type: application/json" \
  -d '{
    "cpuPercent": 45.5,
    "ramPercent": 60.2,
    "hostname": "test-host"
  }'

Stato agenti

Verifica tutti gli agent connessi e il loro stato.

bash
# Lista agenti
curl http://localhost:5000/api/agent

# Alert attivi
curl http://localhost:5000/api/alert\
  ?resolved=false

# Statistiche sistema
curl http://localhost:5000/api/system/stats
