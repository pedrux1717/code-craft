API Reference

Tutti gli endpoint del backend AIROP. Base URL: http://localhost:5000 — Gli endpoint protetti richiedono una sessione autenticata.

Risorse

Authentication System Metrics Process Agents Alerts AI Decisions

Guide

Quick Start

Authentication /api/auth

Autenticazione session-based. Il login crea un cookie HTTP-only con timeout idle di 8 ore. SHA256 password hashing.

POST/api/auth/loginLogin utente

Autentica l'utente. Crea una sessione con cookie HTTP-only. Restituisce username, isAdmin e redirect.

json
{ "username": "admin", "password": "YourPass123" }

// Response
{ "success": true, "user": {
  "id": 1, "username": "admin", "isAdmin": true
}}
POST/api/auth/logoutLogoutAuth

Termina la sessione corrente e invalida il cookie.

GET/api/auth/verifyVerifica sessione

Controlla se la sessione corrente è valida. Usato dal frontend per proteggere le route.

GET/api/auth/is-adminCheck ruolo adminAuth

Restituisce true se l'utente corrente ha ruolo Admin.

POST/api/auth/init-adminCrea primo admin

Crea il primo utente admin. Funziona solo se il DB è vuoto (nessun utente esistente).

json
{ "username": "admin", "password": "SecurePass123!",
  "email": "admin@airop.local" }

System Metrics /api/system

Ricezione e consultazione delle metriche di sistema inviate dagli agent. Tutti gli endpoint richiedono autenticazione.

POST/api/systemInvia metricheAuth

Riceve metriche dall'agent: hostname, CPU%, RAM%, disco, timestamp. Usato da Agent v2.

json
{ "hostname": "server-01",
  "cpuPercent": 45.5,
  "ramPercent": 62.3,
  "diskPercent": 78.1,
  "timestamp": "2026-05-24T10:30:45Z" }
GET/api/system/latestUltime metricheAuth

Ultime metriche da tutti gli host. Usato dalla Dashboard ogni 15 secondi e dall'AI Engine.

GET/api/system/statsStatistiche generaliAuth

Statistiche aggregate: numero metriche, host attivi, medie CPU/RAM. Utile per health check.

GET/api/system/{hostname}Metriche per hostAuth

Storico metriche per un host specifico. Usato dall'AI Engine per analisi su finestre temporali.

Process /api/process

Gestione processi. Endpoint riservati solo agli amministratori.

GET/api/processLista processiAdmin

Lista processi attivi sull'host. Visibile solo agli admin nella pagina Anomaly Detection.

POST/api/process/killKill processoAdmin

Termina un processo per PID. Richiede sessione admin attiva. Restituisce 403 per non-admin.

json
{ "pid": 1234, "reason": "High CPU usage" }

// Admin → 200 OK
{ "success": true, "message": "Process terminated" }

// Non-Admin → 403
{ "success": false, "message": "Admin required" }

Agents /api/agent

Gestione degli agent. Un agent diventa Offline se il heartbeat manca per più di 5 minuti.

POST/api/agent/heartbeatHeartbeatAuth

Segnala che l'agent è attivo. Se l'agent non esiste viene creato. Aggiorna LastHeartbeat e Status=Online.

json
{ "hostname": "prod-01", "agentVersion": "2.0" }
GET/api/agentLista agentAuth

Lista tutti gli agent. Aggiorna automaticamente a Offline se heartbeat mancante > 5 min.

GET/api/agent/{hostname}Dettagli agentAuth

Dettagli e stato di un agent specifico per hostname.

Alerts /api/alert

Alert con tre severità: CRITICAL, WARNING, INFO. Generati dall'AI Engine o manualmente.

POST/api/alertCrea alertAuth

Crea un nuovo alert. Tipicamente chiamato dall'AI Engine su anomalia rilevata.

GET/api/alertLista alertAuth

Tutti gli alert. Query param: ?resolved=false per solo attivi.

GET/api/alert/recentAlert recentiAuth

Alert delle ultime ore. Usato dalla dashboard per il widget alert recenti.

PUT/api/alert/{id}/resolveRisolvi alertAdmin

Marca un alert come risolto. IsResolved=true, ResolvedAt=DateTime.UtcNow.

AI Decisions /api/aidecision

Decisioni AI con workflow di approvazione admin. Nessuna azione viene eseguita senza approvazione esplicita. Status: PENDING → APPROVED/REJECTED → EXECUTED/FAILED.

POST/api/aidecision/generateGenera decisione AIAuth

L'LLM analizza il contesto e genera una raccomandazione con confidence score 0–100%. Salvata come PENDING.

json
{ "hostname": "server-01",
  "alertType": "HIGH_CPU",
  "metrics": { "cpu": 95, "memory": 75 }
}

// Response
{ "id": 42,
  "recommendation": "Kill PID 1234",
  "reasoning": "Process using 60% CPU",
  "confidenceScore": 87,
  "status": "PENDING" }
GET/api/aidecision/pendingDecisioni in attesaAdmin

Lista tutte le decisioni con status PENDING. Usato dalla pagina AI Approvals.

POST/api/aidecision/{id}/approveApprova decisioneAdmin

Admin approva la decisione. Status → APPROVED. L'azione non viene ancora eseguita.

json
{ "notes": "Approved after review" }
POST/api/aidecision/{id}/rejectRifiuta decisioneAdmin

Admin rifiuta la decisione. Status → REJECTED. Loggato nell'AIAuditLog.

POST/api/aidecision/{id}/executeEsegui decisioneAdmin

Esegue una decisione APPROVED. Solo azioni precedentemente approvate possono essere eseguite. Status → EXECUTED/FAILED.

GET/api/aidecision/historyStorico decisioniAdmin

Storico completo di tutte le decisioni. Query param: ?limit=50

GET/api/aidecision/status/enabledStato AIAuth

Verifica se il sistema AI è abilitato. Gestito tramite SystemSettings nel DB.

POST/api/aidecision/status/enabledToggle AIAdmin

Abilita o disabilita il sistema AI. Solo admin.

json
{ "enabled": true }

Quick Start

bash — curl examples
# 1. Crea admin (solo se DB vuoto)
curl -X POST http://localhost:5000/api/auth/init-admin \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"test123","email":"admin@test.com"}'

# 2. Login (salva cookie)
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{"username":"admin","password":"test123"}'

# 3. Statistiche sistema
curl http://localhost:5000/api/system/stats -b cookies.txt

# 4. Genera decisione AI
curl -X POST http://localhost:5000/api/aidecision/generate \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"hostname":"server-01","alertType":"HIGH_CPU","metrics":{"cpu":95}}'

# 5. Kill process (admin only)
curl -X POST http://localhost:5000/api/process/kill \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{"pid":1234,"reason":"High CPU"}'
