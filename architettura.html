// Architettura & Stack

Come è costruito AIROP

Sistema distribuito a 5 livelli: Agent → Backend → MySQL → AI Engine + Automation → Dashboard React. Tutto deployabile su Windows con un singolo START_ALL.bat.

// Deployment

Avvia AIROP

▶ Avvio automatico (Windows)

Avvia tutti i servizi con un singolo script batch.

batch
rem Avvia tutti i servizi
scripts\START_ALL.bat

rem Apre 4 terminali:
rem  1. Backend (Port 5000)
rem  2. React Dashboard (Port 3000)
rem  3. Agent V2
rem  4. AI Engine

rem Stop tutti i servizi
scripts\STOP_ALL.bat

🐳 Docker Compose

Deploy containerizzato con docker-compose.

bash
# Build e avvio
docker-compose \
  -f build/docker-compose.yml \
  up --build

# Env variables
ASPNETCORE_ENVIRONMENT=Production
MYSQL_DATABASE=airop_db
REACT_APP_API_BASE_URL=\
  http://backend:5000/api
// Sicurezza

Security

Implementato

Session Auth

HttpContext.Session con cookie HTTP-only, SameSite=Lax. Timeout idle di 8 ore. SHA256 password hashing con Base64.

Implementato

RBAC

Admin: dashboard completa, kill process, AI approvals. User: solo grafici read-only. Tutti gli endpoint admin sono protetti.

Implementato

CORS

Configurato per localhost:3000 con AllowCredentials per trasmissione cookie sicura.

Implementato

Audit Trail

AIAuditLog: ogni decisione AI, approvazione, rifiuto, esecuzione loggata con userId e timestamp.

Implementato

No Auto-Exec

Nessuna azione AI viene eseguita automaticamente. Tutte le decisioni richiedono esplicita approvazione admin.

Pianificato

HTTPS + JWT

TLS per tutte le comunicazioni e JWT tokens con refresh per produzione. Rate limiting e 2FA in roadmap v2.
