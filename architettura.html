// Deployment

Avvia AIROP

🐳 Docker Compose (Consigliato)

Avvia tutti i componenti con un singolo comando. Ideale per sviluppo e staging.

bash
# Clone e avvio
git clone https://github.com/sorintlab/airop
cd airop
docker-compose up --build

# Dashboard disponibile a:
open http://localhost:3000

⎈ Kubernetes (Production)

Deploy su cluster Kubernetes con HPA per scaling automatico dei pod.

bash
# Apply manifests
kubectl apply -f kubernetes.yaml

# Verifica pods
kubectl get pods -n airop

# Scale backend
kubectl scale deployment airop-backend \
  -n airop --replicas=3
// Sicurezza

Security

Implementato

HTTPS / TLS

Tutte le comunicazioni tra componenti avvengono su HTTPS. Certificati TLS configurati per ambienti production.

Implementato

CORS

CORS configurato lato backend per permettere accesso solo dalle origini autorizzate (dashboard).

Implementato

Input Validation

Validazione su tutti gli endpoint API con risposta standardizzata agli errori. Nessun dato raw arriva al database.

Implementato

Environment Secrets

Credenziali e connection string gestite tramite variabili d'ambiente, mai hardcodate nel codice.

Pianificato

JWT Authentication

Autenticazione token-based per proteggere tutti gli endpoint API. Implementazione prevista in v1.1.

Pianificato

Rate Limiting

Limitazione delle richieste per prevenire abusi degli endpoint pubblici. Previsto insieme a DDoS protection.
