Componente 01

// Agent Data Collection

Agent v2

Il cuore della raccolta dati. Agent Python headless configurabile via file YAML, progettato per girare su qualsiasi server Linux senza interfaccia grafica. Ogni agent monitora il proprio host e invia i dati al backend centrale ogni 5 secondi, insieme a un heartbeat periodico.

  • Raccolta CPU, RAM, disco, rete, processi con psutil
  • Configurazione tramite config.yaml (URL, intervallo, metriche)
  • Heartbeat periodico per segnalare stato Online/Offline
  • Retry automatico su errori di connessione (3 tentativi)
  • Logging strutturato su file airop_agent.log
  • Supporto distribuzione multi-host
Python 3.11+ psutil PyYAML requests
config.yaml
agent:
  poll_interval: 5
  enabled_metrics:
    - cpu
    - memory
    - disk
    - network
    - processes

backend:
  url: http://localhost:5000
  timeout: 10
  retry_attempts: 3

metrics:
  cpu:
    collect_loadavg: true
  memory:
    collect_swap: true
  processes:
    threshold_cpu_percent: 5
    top_count: 10

Componente 02

// Backend Central

REST API C# .NET 8

Il backend è il punto centrale di tutta la piattaforma: riceve metriche dagli agent, le persiste su MSSQL e le espone a dashboard, AI engine e automation engine tramite API REST. Costruito con ASP.NET Core e Entity Framework Core per performance e affidabilità enterprise.

  • 20+ endpoint REST divisi in 4 controller (System, Agent, Alert, Automation)
  • Entity Framework Core con MSSQL Server 2022
  • AutoMapper per la mappatura DTO ↔ Model
  • CORS configurato per permettere accesso dalla dashboard
  • Logging strutturato con Serilog (console + file rolling giornaliero)
  • Swagger UI in ambiente Development
  • Agent offline detection: heartbeat mancante > 5 minuti → status Offline
C# .NET 8 ASP.NET Core Entity Framework MSSQL Serilog

Componente 03

// AI Engine

Anomaly Detection & Prediction

Il modulo di Machine Learning analizza le metriche storiche per rilevare anomalie e prevedere trend. Usa Isolation Forest per identificare valori anomali nelle distribuzioni di CPU e RAM, e Linear Regression per stimare quando una risorsa raggiungerà soglie critiche.

  • Isolation Forest con contamination=0.1 (sklearn)
  • StandardScaler per normalizzazione feature
  • Linear Regression per previsione trend su 24h
  • Generazione automatica di alert CRITICAL/WARNING/INFO
  • Analisi eseguita ogni 5 minuti su tutte le metriche storiche
  • Logging dettagliato in ai_analysis.log
scikit-learn pandas numpy Isolation Forest
anomaly_detector.py
class AIROPAIEngine:
  def __init__(self, backend_url):
    self.scaler = StandardScaler()
    self.anomaly_detector = IsolationForest(
      contamination=0.1,
      random_state=42
    )
    self.metrics_history = {}

  def detect_anomalies(self, metrics):
    X = self.scaler.fit_transform(metrics)
    predictions = self.anomaly_detector.fit_predict(X)
    # -1 = anomalia, 1 = normale
    return predictions == -1

  def predict_trend(self, values):
    model = LinearRegression()
    X = np.arange(len(values)).reshape(-1,1)
    model.fit(X, values)
    return model.coef_[0]  # slope

Componente 04

// Automation Engine

Self-Healing

Il motore di automazione valuta regole configurabili sulle metriche in arrivo e applica azioni correttive automaticamente. Ogni azione è registrata nel database per audit completo. In caso di anomalia non gestibile localmente, viene eseguita l'escalation verso l'amministratore.

  • Tipi azione: KILL_PROCESS, RESTART_SERVICE, SCALE_UP, SCALE_DOWN, CLEAR_CACHE, ESCALATE
  • Regole definite come trigger: metrica + operatore + soglia
  • Esempio: cpu_percent > 80 → kill process più pesante
  • Tracking completo con status: PENDING → EXECUTING → DONE/FAILED
  • Escalation automatica quando nessuna regola risolve il problema
  • Logging su automation.log
Python subprocess Enum REST API
self_healing.py
class ActionType(Enum):
  KILL_PROCESS     = "KILL_PROCESS"
  RESTART_SERVICE  = "RESTART_SERVICE"
  SCALE_UP         = "SCALE_UP"
  SCALE_DOWN       = "SCALE_DOWN"
  CLEAR_CACHE      = "CLEAR_CACHE"
  ESCALATE         = "ESCALATE"

class AutomationRule:
  def check_trigger(self, metrics):
    metric = self.trigger['metric']
    op     = self.trigger['operator']
    thresh = self.trigger['threshold']
    value  = metrics.get(metric, 0)

    # Valuta: cpu_percent > 80
    if op == '>': return value > thresh

L'interfaccia di monitoraggio costruita con React 18 e Material-UI. Aggiornaautomaticamente ogni 10 secondi, mostrando metriche in tempo reale, alert attivi, stato degli agent e storico delle azioni di automazione. Tutto con grafici interattivi e navigazione tra sezioni.

  • Dashboard principale: LineChart CPU e RAM sugli ultimi 20 campioni
  • PieChart distribuzione alert per severità (CRITICAL/WARNING/INFO)
  • Pagina Agents: lista con status Online/Offline e ultimo heartbeat
  • Pagina Alerts: filtro per severità e risoluzione diretta
  • Pagina Anomaly Detection: risultati AI Engine in tempo reale
  • Aggiornamento automatico ogni 10 secondi con setInterval
