AIROP —
il futuro
dell'IT aziendale.

Una piattaforma software che trasforma l'infrastruttura IT da reattiva a predittiva. Monitora, rileva anomalie con AI, propone azioni correttive e le esegue solo dopo approvazione admin — con audit trail completo.

// Perché esiste

Gli Obiettivi

01

Monitoraggio Real-Time

Raccogliere metriche di sistema ogni 5 secondi da tutti gli host della rete, con dashboard che si aggiorna ogni 15 secondi.

02

AI con Supervisione Umana

L'AI rileva anomalie e propone azioni, ma nessuna azione viene eseguita senza esplicita approvazione dell'amministratore.

03

Sicurezza Enterprise

Autenticazione session-based, role-based access control (Admin/User), audit log completo e password hashing SHA256.

Componente 01

// Data Collection

Agent v2

Agent Python headless configurabile via YAML. Gira su ogni host della rete, raccoglie metriche e le invia al backend ogni 5 secondi. Invia anche heartbeat periodici: se mancante, l'agent risulta Offline.

  • CPU, RAM, disco, rete, processi con psutil
  • Configurazione via config.yaml
  • Heartbeat per status Online/Offline
  • Retry automatico su errori (3 tentativi)
  • Post a http://localhost:5000/api/system
Python 3.11+psutilPyYAMLrequests
config.yaml
agent:
  poll_interval: 5
  enabled_metrics:
    - cpu
    - memory
    - disk
    - network
    - processes

backend:
  url: http://localhost:5000
  timeout: 10
  retry_attempts: 3

Componente 03

// AI Engine

Anomaly Detection + AI Decisions

Due livelli AI: Isolation Forest (sklearn) per rilevare anomalie nelle metriche ogni 5 minuti, e un LLM (Ollama/Mistral o OpenAI) che analizza il contesto e genera raccomandazioni con confidence score.

  • Isolation Forest contamination=0.1
  • StandardScaler per normalizzazione
  • LLM: Ollama (Mistral, gratuito/locale) o OpenAI
  • Ogni decisione salvata come PENDING nel DB
  • Nessuna esecuzione senza approvazione admin
sklearnOllamaOpenAIpandas
AI Decision Flow
# 1. Alert rilevato
Alert triggered: CPU_HIGH (95%)

# 2. AI genera raccomandazione
POST /api/aidecision/generate
→ recommendation: "Kill PID 1234"
→ reasoning: "Process using 60% CPU"
→ confidence: 87%
→ status: PENDING

# 3. Admin rivede e approva
POST /api/aidecision/{id}/approve

# 4. Admin esegue
POST /api/aidecision/{id}/execute
→ status: EXECUTED
→ audit log aggiornato

Componente 04

// Automation Engine

Self-Healing

Il motore di automazione valuta regole configurabili ogni 10 minuti. Può killare processi, riavviare servizi, svuotare cache. Tutte le azioni richiedono approvazione admin ed è mantenuto un audit trail completo.

  • Tipi: KILL_PROCESS, RESTART_SERVICE, SCALE_UP/DOWN, CLEAR_CACHE, ESCALATE
  • Ogni azione: PENDING → APPROVED → EXECUTED/FAILED
  • Intervallo analisi: ogni 10 minuti
  • Kill process diretto disponibile da dashboard (admin)
PythonsubprocessAudit Trail
self_healing.py
class ActionType(Enum):
  KILL_PROCESS    = "KILL_PROCESS"
  RESTART_SERVICE = "RESTART_SERVICE"
  SCALE_UP        = "SCALE_UP"
  CLEAR_CACHE     = "CLEAR_CACHE"
  ESCALATE        = "ESCALATE"

class AutomationRule:
  def check_trigger(self, metrics):
    value = metrics.get(self.metric, 0)
    if self.op == '>':
      return value > self.threshold
    # cpu_percent > 80 → kill

Componente 06

// AI Decision System

Approval Workflow

Il sistema di AI Decision è il cuore dell'autonomia supervisionata di AIROP. L'LLM analizza alert e metriche, genera raccomandazioni con confidence score 0–100%, e le salva come PENDING. Solo l'admin può approvarle ed eseguirle.

  • Supporto Ollama (Mistral, gratuito e locale) e OpenAI
  • Ogni decisione: hostname, tipo, reasoning, confidence
  • Status: PENDING → APPROVED/REJECTED → EXECUTED/FAILED
  • Dashboard /ai-approvals con statistiche e history
  • AIAuditLog: chi ha approvato, quando, risultato
